### Reporting Bugs üêõ

If you find a bug, please create an issue with:
- A clear, descriptive title
- Steps to reproduce the problem
- Expected vs actual behavior
- Your R version and package versions
- Screenshots if applicable
- Any relevant error messages

### Suggesting Enhancements üí°

Have ideas for new features or improvements? We'd love to hear them!

Please create an issue describing:
- What problem does it solve?
- How would it work?
- Any examples from other projects?
- Why would this be useful to users?

### Pull Requests üöÄ

We welcome pull requests! Here's how:

1. **Fork the repo** and create your branch from `main`
2. **Make your changes** following the guidelines below
3. **Test your code** - make sure the R script still runs
4. **Update documentation** if you changed functionality
5. **Write a clear commit message** describing what you did
6. **Submit a pull request** with a comprehensive description

## Development Guidelines

### For R Code Changes

- Follow R style conventions (use tidyverse style guide)
- Comment your code, especially complex logic
- Keep functions focused on single tasks
- Test with different data subsets if possible
- Ensure existing visualizations still generate correctly

### For Documentation

- Use clear, concise language
- Include code examples where helpful
- Keep formatting consistent with existing docs
- Check for spelling and grammar
- Update the README if you add new features

### Commit Message Guidelines

Write clear commit messages that explain what and why:

**Good examples:**
- `Fix: Correct clustering algorithm initialization`
- `Add: New filter for institution accreditation status`
- `Docs: Update installation instructions for Mac users`
- `Improve: Optimize data preprocessing for large datasets`

**Less helpful:**
- `fix bug`
- `update code`
- `changes`

## Code Review Process

1. Maintainers will review your PR within a few days
2. We may suggest changes or improvements
3. Once approved, your PR will be merged
4. You'll be added to the contributors list!

## What We're Looking For

Current areas where contributions would be especially valuable:

- **Additional visualizations** showing different data relationships
- **Performance improvements** for large datasets
- **New recommendation features** (e.g., cost-based filtering, program-specific recommendations)
- **Better documentation** with more examples
- **Bug fixes** for any issues you encounter
- **UI/UX improvements** for the Shiny application
- **Testing** - help us add unit tests

## Questions?
